🚀 GitHub Repo Setup

Terminal-inspired CV site for capybaraco.xyz

Step 1: Create a GitHub Account

If you don't already have a GitHub account, create one now:

→ Go to GitHub Signup

Register with your main email address, verify it, and note your username — you'll need it below.

Step 2: Install the gh CLI

Select your operating system and run the appropriate command:

brew install gh
sudo apt update && sudo apt install gh
sudo dnf install gh
choco install gh

Once installed, verify it works:

gh --version

Step 3: Authenticate gh with GitHub

Run this command and follow the interactive prompts:

gh auth login

When prompted:

  • "What is your preferred protocol for Git operations?" → Choose HTTPS
  • "Authenticate Git with your GitHub credentials?" → Choose Yes
  • "How would you like to authenticate GitHub CLI?" → Choose Login with a web browser

A browser window will open for you to authorize the CLI. Once complete, verify:

gh auth status

You should see your GitHub username and a checkmark (✓).

Step 4: Set Your Local Git Identity

Configure git with your name and email (must match your GitHub account):

git config --global user.name "Aaron Paterson" git config --global user.email "aaron.paterson@capybaraco.co.uk"

Verify:

git config --global user.name git config --global user.email

Step 5: Initialize Git & Create the Remote Repo

Navigate to your project folder:

cd /home/sleepysleepy/website-cv

Initialize git:

git init

Create the remote repo on GitHub (public):

gh repo create capybaraco-cv --public --remote=origin --source=. --remote-name=origin --push=false

Verify the remote is set:

git remote -v

You should see two lines showing origin with your GitHub username.

Step 6 (Optional): Set the Repo About Link

Once the repo is created, visit your repo on GitHub and fill in the About section (top right):

  • Description: "Terminal-inspired CV site"
  • Website: https://www.capybaraco.xyz
  • Topics: portfolio, cv, react, vite, terminal

View Your Repo

✓ You're Ready!

  • gh auth status shows you're logged in
  • git remote -v shows your remote is set
  • Your repo is live on GitHub (public)
